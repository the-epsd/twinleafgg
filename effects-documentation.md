# Pokémon TCG Server Effects Documentation

## Introduction

The effects system in the Pokémon TCG server is the backbone for managing game actions, events, and their consequences. It is designed to be flexible, allowing various game mechanics and card abilities to interact with and modify standard game flow.

At the core of this system is the `Effect` interface. Each distinct game event or action that can be influenced or observed is represented as an object implementing this interface. These effects are typically created and then dispatched through a central mechanism in the game engine (e.g., a `Store` or an effect resolver). Other parts of the system, such as card abilities or game phase handlers, can listen for specific effect types and react to them.

A key feature of the `Effect` interface is the `preventDefault` property. If a handler for an effect sets `preventDefault` to `true`, it signals that the default outcome of that effect should be suppressed. This allows, for example, an ability to stop a Pokémon from being affected by an attack or to prevent a player from drawing a card.

This document details the various types of effects defined in the system, their purposes, properties, and how they contribute to the game's logic.

## I. Core Effect Definition (`effect.ts`)

This file lays the groundwork for the entire effects system by defining the fundamental `Effect` interface.

### 1. `Effect` Interface
    - **Purpose:** The `Effect` interface is the basic contract that all game effects must adhere to. It provides a common structure for identifying and controlling the behavior of game events.
    - **Properties:**
        - `type: string` (readonly): This property serves as a unique identifier for the kind of effect. It's typically populated by values from specific enums (e.g., `GameEffects`, `AttackEffects`). The game engine uses this type to route the effect to appropriate handlers or reducers.
        - `preventDefault: boolean`: A crucial flag that determines whether the default action associated with this effect should proceed. Initially `false`, any part of the system handling the effect can set this to `true` to indicate that the effect's default consequence has been handled, modified, or should be entirely stopped.

## II. Effect Categories

### A. Game Effects (`game-effects.ts`)

This file defines a collection of general game events and actions that represent core mechanics. These effects are fundamental to the game's operation and cover a wide range of player actions and game state changes.

#### 1. `GameEffects` Enum
    - **Purpose:** Provides a comprehensive list of string constants that identify the types of general game effects.
    - **Values:**
        - `RETREAT_EFFECT`: Represents a Pokémon retreating from the active position.
        - `USE_ATTACK_EFFECT`: An effect triggered when a player declares the use of an attack.
        - `USE_STADIUM_EFFECT`: An effect triggered when a Stadium card's effect is used.
        - `USE_POWER_EFFECT`: An effect triggered when a player uses a Pokémon Power (Ability).
        - `POWER_EFFECT`: Represents the actual execution or impact of a Pokémon Power.
        - `ATTACK_EFFECT`: Represents the execution phase of an attack, including damage calculation and other attack consequences.
        - `KNOCK_OUT_EFFECT`: An effect triggered when a Pokémon is Knocked Out.
        - `HEAL_EFFECT`: Represents healing damage from a Pokémon.
        - `EVOLVE_EFFECT`: An effect triggered when a Pokémon evolves.
        - `DRAW_PRIZES_EFFECT`: An effect for when a player draws Prize cards.
        - `MOVE_CARDS_EFFECT`: A generic effect for moving cards between different game zones (e.g., deck to hand, discard to deck).
        - `EFFECT_OF_ABILITY_EFFECT`: Represents the effect specifically generated by an Ability (distinct from just using the power).
        - `SPECIAL_ENERGY_EFFECT`: An effect related to the special properties or actions of a Special Energy card.

#### 2. Effect Classes:

    - **`RetreatEffect` Effect**
        - `type`: `GameEffects.RETREAT_EFFECT`
        - **Purpose:** Manages the process of a player's Active Pokémon retreating to the Bench.
        - **Constructor Parameters:**
            - `player: Player`: The player performing the retreat.
            - `benchIndex: number`: The index of the Pokémon on the Bench that will become active.
        - **Key Properties:**
            - `player: Player`: The player initiating the retreat.
            - `benchIndex: number`: The index of the benched Pokémon chosen to become active.
            - `ignoreStatusConditions: boolean`: If `true`, status conditions do not prevent this retreat (default `false`).
            - `moveRetreatCostTo: CardList`: The destination for the discarded energy cards used to pay the retreat cost (defaults to `player.discard`).
        - **Example Scenario:** Triggered when a player chooses the "Retreat" action during their turn. Handlers for this effect would manage discarding energy for the retreat cost, moving the Pokémon, and promoting the new active Pokémon.

    - **`UsePowerEffect` Effect**
        - `type`: `GameEffects.USE_POWER_EFFECT`
        - **Purpose:** Represents a player's intent to use a Pokémon's Power (Ability).
        - **Constructor Parameters:**
            - `player: Player`: The player using the Power.
            - `power: Power`: The Power object being used.
            - `card: PokemonCard`: The Pokémon card that has the Power.
            - `target: CardTarget`: Optional target for the power, if applicable.
        - **Key Properties:** (as above)
        - **Example Scenario:** Triggered when a player activates an Ability on one of their Pokémon. This effect allows other cards or game rules to potentially prevent the Power's usage by setting `preventDefault` to `true`.

    - **`UseTrainerPowerEffect` Effect**
        - `type`: `GameEffects.USE_POWER_EFFECT` (Note: Same type as `UsePowerEffect`)
        - **Purpose:** Represents a player's intent to use a Power originating from a Trainer card (e.g., an Item with an Ability-like effect, or a Pokémon Tool granting an ability).
        - **Constructor Parameters:**
            - `player: Player`: The player using the Power.
            - `power: Power`: The Power object (defined on the Trainer card).
            - `card: TrainerCard`: The Trainer card that has the Power.
            - `target: CardTarget`: Optional target for the power.
        - **Key Properties:** (as above)

    - **`UseEnergyPowerEffect` Effect**
        - `type`: `GameEffects.USE_POWER_EFFECT` (Note: Same type as `UsePowerEffect`)
        - **Purpose:** Represents a player's intent to use a Power originating from an Energy card (typically Special Energy cards).
        - **Constructor Parameters:**
            - `player: Player`: The player using the Power.
            - `power: Power`: The Power object (defined on the Energy card).
            - `card: EnergyCard`: The Energy card that has the Power.
            - `target: CardTarget`: Optional target for the power.
        - **Key Properties:** (as above)

    - **`PowerEffect` Effect**
        - `type`: `GameEffects.POWER_EFFECT`
        - **Purpose:** Represents the actual resolution or impact of a Pokémon's Power after it has been successfully used.
        - **Constructor Parameters:**
            - `player: Player`: The player whose Power is resolving.
            - `power: Power`: The Power object.
            - `card: PokemonCard`: The Pokémon card that used the Power.
            - `target?: PokemonCardList`: Optional target of the Power.
        - **Key Properties:** (as above)
        - **Example Scenario:** After `UsePowerEffect` is processed and not prevented, this effect would be dispatched to carry out the Power's text (e.g., drawing cards, placing damage counters).

    - **`TrainerPowerEffect` Effect**
        - `type`: `GameEffects.POWER_EFFECT` (Note: Same type as `PowerEffect`)
        - **Purpose:** Represents the actual resolution of a Power originating from a Trainer card.
        - **Constructor Parameters:**
            - `player: Player`: The player whose Trainer card's Power is resolving.
            - `power: Power`: The Power object.
            - `card: TrainerCard`: The Trainer card.
        - **Key Properties:** (as above)

    - **`UseAttackEffect` Effect**
        - `type`: `GameEffects.USE_ATTACK_EFFECT`
        - **Purpose:** Signifies a player declaring an attack with their Active Pokémon.
        - **Constructor Parameters:**
            - `player: Player`: The player declaring the attack.
            - `attack: Attack`: The attack being used.
        - **Key Properties:**
            - `player: Player`: The attacking player.
            - `attack: Attack`: The specific attack.
            - `source: PokemonCardList`: The Pokémon using the attack (defaults to `player.active`).
        - **Example Scenario:** Triggered when a player selects an attack during their turn. Effects that prevent attacks (e.g., due to Special Conditions, Abilities) would handle this.

    - **`UseStadiumEffect` Effect**
        - `type`: `GameEffects.USE_STADIUM_EFFECT`
        - **Purpose:** Represents the activation of a Stadium card's ongoing effect.
        - **Constructor Parameters:**
            - `player: Player`: The player whose turn it is (Stadium effects often affect both players).
            - `stadium: Card`: The Stadium card in play.
        - **Key Properties:** (as above)

    - **`AttackEffect` Effect**
        - `type`: `GameEffects.ATTACK_EFFECT`
        - **Purpose:** Manages the entire process of an attack resolving, from calculating damage to applying secondary effects. This is a major effect that often triggers many sub-effects (defined in `attack-effects.ts`).
        - **Constructor Parameters:**
            - `player: Player`: The attacking player.
            - `opponent: Player`: The defending player.
            - `attack: Attack`: The attack being resolved.
        - **Key Properties:**
            - `player: Player`, `opponent: Player`, `attack: Attack`
            - `damage: number`: The base damage of the attack, can be modified by handlers.
            - `ignoreWeakness: boolean`: If `true`, weakness is not applied.
            - `ignoreResistance: boolean`: If `true`, resistance is not applied.
            - `source: PokemonCardList`: The attacking Pokémon (defaults to `player.active`).
            - `target: any`: The target of the attack (usually `opponent.active`).
            - `invisibleTentacles?: boolean`: A specific flag for a certain attack type.
        - **Example Scenario:** Dispatched after `UseAttackEffect` if the attack was not prevented. This effect will then trigger further effects like `ApplyWeaknessEffect`, `DealDamageEffect`, etc.

    - **`KnockOutEffect` Effect**
        - `type`: `GameEffects.KNOCK_OUT_EFFECT`
        - **Purpose:** Handles the event of a Pokémon being Knocked Out.
        - **Constructor Parameters:**
            - `player: Player`: The player whose Pokémon is Knocked Out (the "owner" of the KO'd Pokémon).
            - `target: PokemonCardList`: The PokémonCardList that is Knocked Out.
        - **Key Properties:**
            - `player: Player`, `target: PokemonCardList`
            - `prizeCount: number`: The number of Prize cards the opponent takes (default 1, can be modified).
            - `prizeDestination?: CardList`: Where the prize cards are moved to (usually opponent's hand).
            - `isLostCity: boolean`: Flag if Lost City stadium is in play, affecting where KO'd Pokémon go.
        - **Example Scenario:** Triggered when a Pokémon's damage meets or exceeds its HP.

    - **`KnockOutAttackEffect` Effect**
        - `type`: `GameEffects.KNOCK_OUT_EFFECT` (Note: Same type as `KnockOutEffect`)
        - **Purpose:** A specialized version of `KnockOutEffect` specifically when the KO is the result of an attack. It carries the `attack` context.
        - **Constructor Parameters:**
            - `player: Player`: The player whose Pokémon is Knocked Out.
            - `target: PokemonCardList`: The KO'd Pokémon.
            - `attack: Attack`: The attack that caused the KO.
        - **Key Properties:** (as above, plus `attack`)

    - **`HealEffect` Effect**
        - `type`: `GameEffects.HEAL_EFFECT`
        - **Purpose:** Represents the removal of damage counters from a Pokémon.
        - **Constructor Parameters:**
            - `player: Player`: The player initiating the heal (can be self or opponent through an attack).
            - `target: PokemonCardList`: The Pokémon being healed.
            - `damage: number`: The amount of damage to heal.
        - **Key Properties:** (as above)
        - **Example Scenario:** Playing a Potion, or an attack that has a healing component.

    - **`EvolveEffect` Effect**
        - `type`: `GameEffects.EVOLVE_EFFECT`
        - **Purpose:** Manages a Pokémon evolving into a higher stage.
        - **Constructor Parameters:**
            - `player: Player`: The player evolving a Pokémon.
            - `target: PokemonCardList`: The Pokémon that is evolving.
            - `pokemonCard: PokemonCard`: The Pokémon card being played to evolve.
        - **Key Properties:**
            - `player: Player`, `target: PokemonCardList`, `pokemonCard: PokemonCard`
            - `darkestImpulseSV?: boolean`: Specific flag for a particular evolution-related ability.
        - **Example Scenario:** A player plays an evolution card from their hand onto one of their Basic or Stage 1 Pokémon.

    - **`DrawPrizesEffect` Effect**
        - `type`: `GameEffects.DRAW_PRIZES_EFFECT`
        - **Purpose:** Handles a player taking Prize cards.
        - **Constructor Parameters:**
            - `player: Player`: The player drawing the prizes.
            - `prizes: CardList[]`: An array of CardLists, each representing a Prize card taken.
            - `destination: CardList`: Where the taken Prize cards are moved (usually the player's hand).
        - **Key Properties:** (as above)
        - **Example Scenario:** Triggered after an opponent's Pokémon is Knocked Out and the `KnockOutEffect` determines prize count.

    - **`MoveCardsEffect` Effect**
        - `type`: `GameEffects.MOVE_CARDS_EFFECT`
        - **Purpose:** A versatile effect for moving one or more cards from a source location to a destination location.
        - **Constructor Parameters:**
            - `source: CardList | PokemonCardList`: The starting location of the cards.
            - `destination: CardList | PokemonCardList`: The target location for the cards.
            - `options: object`: An object containing optional parameters:
                - `cards?: Card[]`: Specific cards to move. If not provided, `count` might be used.
                - `count?: number`: Number of cards to move (e.g., top X cards from deck).
                - `toTop?: boolean`: If `true`, move cards to the top of the destination.
                - `toBottom?: boolean`: If `true`, move cards to the bottom of the destination.
                - `skipCleanup?: boolean`: If `true`, skip certain cleanup operations that might normally occur after moving cards.
                - `sourceCard?: Card`: The card that initiated this move (e.g., a Trainer card).
        - **Key Properties:** (as above)
        - **Example Scenario:** Drawing cards from the deck, discarding cards from hand, moving Pokémon between active/bench.

    - **`EffectOfAbilityEffect` Effect**
        - `type`: `GameEffects.EFFECT_OF_ABILITY_EFFECT`
        - **Purpose:** Specifically denotes that an effect is a direct consequence of an Ability (Pokémon Power). This helps differentiate Ability-driven effects from other game actions.
        - **Constructor Parameters:**
            - `player: Player`: The player whose Ability is causing the effect.
            - `power: Power`: The Power object.
            - `card: PokemonCard`: The Pokémon card with the Ability.
            - `target?: PokemonCardList`: Optional target of the Ability's effect.
        - **Key Properties:** (as above)

    - **`SpecialEnergyEffect` Effect**
        - `type`: `GameEffects.SPECIAL_ENERGY_EFFECT`
        - **Purpose:** Represents an effect that is specifically tied to a Special Energy card being in play or its abilities.
        - **Constructor Parameters:**
            - `player: Player`: The player controlling the Special Energy card.
            - `card: EnergyCard`: The Special Energy card itself.
            - `attachedTo: PokemonCardList`: The Pokémon the Special Energy is attached to.
            - `exemptFromOpponentsSpecialEnergyBlockingAbility: boolean` (default `false`): Flag if this energy's effect bypasses certain blocking abilities (e.g. Temple of Sinnoh).
        - **Key Properties:** (as above)
        - **Example Scenario:** A Double Colorless Energy providing two Colorless energy, or a Rainbow Energy having an effect when attached.

### B. Attack Effects (`attack-effects.ts`)

This file details the sequence of sub-effects that occur during the resolution of a Pokémon's attack. These effects break down the attack process into granular steps.

#### 1. `AttackEffects` Enum
    - **Purpose:** Lists the types of effects specific to the attack resolution phase.
    - **Values:**
        - `APPLY_WEAKNESS_EFFECT`: For applying weakness and resistance to damage.
        - `DEAL_DAMAGE_EFFECT`: Represents the point where damage is about to be dealt, after calculations like weakness/resistance.
        - `PUT_DAMAGE_EFFECT`: The actual placement of damage counters on a Pokémon.
        - `KNOCK_OUT_OPPONENT_EFFECT`: A specific effect for when an opponent is KO'd by damage (seems to overlap with `DEAL_DAMAGE_EFFECT` purpose, might be for specific triggers).
        - `AFTER_DAMAGE_EFFECT`: For effects that trigger after damage has been dealt.
        - `PUT_COUNTERS_EFFECT`: For placing generic counters (not necessarily damage counters).
        - `DISCARD_CARD_EFFECT`: For effects that require discarding cards as part of an attack.
        - `CARDS_TO_HAND_EFFECT`: For effects that move cards to hand as part of an attack.
        - `GUST_OPPONENT_BENCH_EFFECT`: For effects that switch one of the opponent's Benched Pokémon to the Active spot.
        - `ADD_MARKER_EFFECT`: For adding specific markers to a Pokémon.
        - `ADD_SPECIAL_CONDITIONS_EFFECT`: For applying Special Conditions as part of an attack.
        - `MOVED_TO_ACTIVE_BONUS_EFFECT`: For effects that grant a bonus if a Pokémon just moved to the active spot.
        - `LOST_ZONED_CARDS_EFFECT`: For effects that move cards to the Lost Zone as part of an attack.


#### 2. `AbstractAttackEffect` Class
    - **Purpose:** Serves as a base abstract class for most attack-related sub-effects. It ensures that all attack sub-effects carry common contextual information.
    - **Key Properties (inherited by subclasses):**
        - `attackEffect: AttackEffect`: The parent `AttackEffect` (from `game-effects.ts`) that this sub-effect is part of.
        - `attack: Attack`: The specific attack being resolved.
        - `player: Player`: The attacking player.
        - `opponent: Player`: The defending player.
        - `target: PokemonCardList`: The Pokémon being targeted by the attack (usually `opponent.active`).
        - `source: PokemonCardList`: The Pokémon performing the attack (usually `player.active`).
        - `preventDefault: boolean`: Standard effect property.

#### 3. Effect Classes:

    - **`ApplyWeaknessEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.APPLY_WEAKNESS_EFFECT`
        - **Purpose:** Calculates and applies damage modifications due to Weakness and Resistance.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The initial damage amount before Weakness/Resistance.
        - **Key Properties:**
            - `damage: number`: The damage amount, which will be modified by this effect's logic based on the target's Weakness and Resistance.
            - `ignoreResistance: boolean`: If true, resistance is not applied.
            - `ignoreWeakness: boolean`: If true, weakness is not applied.

    - **`DealDamageEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.DEAL_DAMAGE_EFFECT`
        - **Purpose:** Represents the step where the calculated damage (after Weakness/Resistance and other modifications) is about to be finalized and dealt to the target.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The final calculated damage to be dealt.
        - **Key Properties:**
            - `damage: number`: The amount of damage to be dealt.
            - `damageIncreased: boolean`: Flag indicating if the damage was increased by any effects.

    - **`PutDamageEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.PUT_DAMAGE_EFFECT`
        - **Purpose:** Represents the physical placement of damage counters on the target Pokémon.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The amount of damage for which counters should be placed.
        - **Key Properties:**
            - `damage: number`: The damage value.
            - `damageReduced: boolean`: Flag indicating if damage was reduced.
            - `damageIncreased: boolean`: Flag indicating if damage was increased (seems to be misnamed in original code, should be `true` by default if damage > 0).
            - `wasKnockedOutFromFullHP: boolean`: Flag if the Pokémon was KO'd from full HP by this damage.
            - `weaknessApplied: boolean`: Flag indicating if weakness was applied in damage calculation.

    - **`AfterDamageEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.AFTER_DAMAGE_EFFECT`
        - **Purpose:** An effect that triggers after damage has been successfully dealt by an attack. Used for abilities or attack clauses that say "After dealing damage...".
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The actual damage that was dealt.
        - **Key Properties:**
            - `damage: number`: The damage that was just dealt.

    - **`PutCountersEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.PUT_COUNTERS_EFFECT`
        - **Purpose:** For placing generic counters (e.g., specific named counters from an attack effect, not necessarily damage counters) on a Pokémon.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The number of counters to place (property name `damage` might be misleading if they aren't damage counters).
        - **Key Properties:**
            - `damage: number`: Number of counters.

    - **`KOEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.PUT_DAMAGE_EFFECT` (Note: Type seems to be a copy-paste error from `PutDamageEffect`, should likely be its own type or align with `GameEffects.KNOCK_OUT_EFFECT` related types if it's for KOs).
        - **Purpose:** Intended to handle Knock Outs resulting from attack damage.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The damage that resulted in the KO.
        - **Key Properties:**
            - `damage: number`: Damage dealt.
            - `damageReduced: boolean`: If damage was reduced.
            - `wasKnockedOutFromFullHP: boolean`: If KO'd from full HP.

    - **`GustOpponentBenchEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.GUST_OPPONENT_BENCH_EFFECT`
        - **Purpose:** Handles effects that force an opponent to switch their Active Pokémon with one from their Bench, or allow the attacker to choose one.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `target: PokemonCardList`: The specific Benched Pokémon that will be moved to the Active spot.
        - **Key Properties:**
            - `target: PokemonCardList`: The benched Pokémon to be made active.

    - **`KnockOutOpponentEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.DEAL_DAMAGE_EFFECT` (Note: Type seems to be a copy-paste error, likely intended to be related to KO)
        - **Purpose:** Similar to `KOEffect`, this seems to handle the Knock Out of an opponent's Pokémon as a result of damage. The distinction from `KOEffect` is unclear without further context on usage.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The damage dealt.
        - **Key Properties:**
            - `damage: number`.

    - **`DiscardCardsEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.DISCARD_CARD_EFFECT`
        - **Purpose:** Manages the discarding of cards (e.g., Energy) as a cost or effect of an attack.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `cards: Card[]`: An array of cards to be discarded.
        - **Key Properties:**
            - `cards: Card[]`: The cards designated for discard.

    - **`LostZoneCardsEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.LOST_ZONED_CARDS_EFFECT`
        - **Purpose:** Handles moving cards to the Lost Zone as an effect of an attack.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `cards: Card[]`: An array of cards to be moved to the Lost Zone.
        - **Key Properties:**
            - `cards: Card[]`: The cards to be Lost Zoned.

    - **`CardsToHandEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.DISCARD_CARD_EFFECT` (Note: Type seems to be a copy-paste error, should be `CARDS_TO_HAND_EFFECT`).
        - **Purpose:** Manages moving cards to a player's hand as an effect of an attack (e.g., returning Energy from discard to hand).
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `cards: Card[]`: An array of cards to be moved to the hand.
        - **Key Properties:**
            - `cards: Card[]`: The cards to be moved.

    - **`AddMarkerEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.ADD_MARKER_EFFECT`
        - **Purpose:** Places a named marker on a Pokémon (e.g., "Burn marker", "Poison marker", or custom markers from specific attacks).
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `markerName: string`: The name of the marker to add.
            - `markerSource: Card`: The card that is the source of this marker.
        - **Key Properties:** (as above)

    - **`AddSpecialConditionsEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.ADD_SPECIAL_CONDITIONS_EFFECT`
        - **Purpose:** Applies one or more Special Conditions to the target Pokémon.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `specialConditions: SpecialCondition[]`: An array of Special Conditions to apply (e.g., `[SpecialCondition.POISONED, SpecialCondition.CONFUSED]`).
        - **Key Properties:**
            - `specialConditions: SpecialCondition[]`: The conditions to apply.
            - `poisonDamage?: number`: If Poison is applied, this can specify the amount of damage it deals (overriding default).

    - **`RemoveSpecialConditionsEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.ADD_SPECIAL_CONDITIONS_EFFECT` (Note: Type should be distinct, e.g., `REMOVE_SPECIAL_CONDITIONS_EFFECT`).
        - **Purpose:** Removes one or more SpecialConditions from the target Pokémon.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `specialConditions: SpecialCondition[] | undefined`: An array of Special Conditions to remove. If `undefined`, removes all standard Special Conditions.
        - **Key Properties:**
            - `specialConditions: SpecialCondition[]`: The conditions to remove.

    - **`HealTargetEffect` Effect** (extends `AbstractAttackEffect`)
        - `type`: `AttackEffects.ADD_MARKER_EFFECT` (Note: Type should be distinct, e.g., `HEAL_TARGET_EFFECT`).
        - **Purpose:** Heals damage from the target Pokémon as part of an attack's effect.
        - **Constructor Parameters:**
            - `base: AttackEffect`: The parent `AttackEffect`.
            - `damage: number`: The amount of damage to heal.
        - **Key Properties:**
            - `damage: number`: Amount to heal.

### C. Check Effects (`check-effects.ts`)

This file contains effects used to query or verify various aspects of the game state, Pokémon statistics, or card properties. These effects typically gather information rather than directly altering the game state, though their `preventDefault` property can still be used to influence subsequent game logic.

#### 1. `CheckEffects` Enum
    - **Purpose:** Lists the types of checks that can be performed.
    - **Values:**
        - `CHECK_HP_EFFECT`: Checks the Hit Points of a Pokémon.
        - `CHECK_PRIZES_COUNT_EFFECT`: Checks the number of Prize cards a player has left.
        - `CHECK_POKEMON_STATS_EFFECT`: Checks a Pokémon's stats like Weakness and Resistance.
        - `CHECK_POKEMON_POWERS_EFFECT`: Checks the Powers (Abilities) a Pokémon has.
        - `CHECK_POKEMON_ATTACKS_EFFECT`: Checks the attacks a Pokémon can use (including those from Tools).
        - `CHECK_POKEMON_TYPE_EFFECT`: Checks the elemental type(s) of a Pokémon.
        - `CHECK_RETREAT_COST_EFFECT`: Checks the energy cost for a Pokémon to retreat.
        - `CHECK_ATTACK_COST_EFFECT`: Checks the energy cost for a Pokémon to use a specific attack.
        - `CHECK_ENOUGH_ENERGY_EFFECT`: Checks if enough energy is attached to a Pokémon to pay for a cost (e.g., for an attack or retreat), and what types of energy are provided.
        - `CHECK_POKEMON_PLAYED_TURN_EFFECT`: Checks which turn a Pokémon was put into play.
        - `CHECK_TABLE_STATE_EFFECT`: Checks general table state like bench sizes.
        - `ADD_SPECIAL_CONDITIONS_EFFECT`: (Name seems misplaced in "Check" effects, as it implies adding. It's used in `AddSpecialConditionsPowerEffect` to *check* and potentially modify how SCs are applied by a power).

### D. Game Phase Effects (`game-phase-effects.ts`)

This file defines effects that correspond to specific moments or transitions within the game's turn structure.

#### 1. `GamePhaseEffects` Enum
    - **Purpose:** Lists types of effects related to game phases.
    - **Values:**
        - `BEGIN_TURN_EFFECT`: Triggered at the very start of a player's turn.
        - `END_TURN_EFFECT`: Triggered at the end of a player's turn, before passing to the opponent.
        - `WHO_BEGINS_EFFECT`: Used during setup to determine which player goes first.
        - `BETWEEN_TURNS_EFFECT`: Triggered during the Pokémon Checkup step between turns (handles poison, burn, sleep/paralysis checks).
        - `CHOOSE_STARTING_POKEMON_EFFECT`: Used during setup when players choose their starting Active and Benched Pokémon.
        - `DREW_TOPDECK_EFFECT`: Triggered after a player draws their card for the turn.
        - `CHOOSE_PRIZE_EFFECT`: Triggered when a player needs to choose Prize cards after a KO.
        - `AFTER_ATTACK_EFFECT`: Triggered immediately after an attack has fully resolved, before Special Conditions are checked for recovery in the end-of-turn sequence.

#### 2. Effect Classes:

    - **`BeginTurnEffect` Effect**
        - `type`: `GamePhaseEffects.BEGIN_TURN_EFFECT`
        - **Purpose:** Signals the start of a player's turn. This is typically when "At the start of your turn..." abilities trigger.

            - `player: Player`: The player whose turn is beginning.
        - **Key Properties:** `player: Player`.

    - **`DrewTopdeckEffect` Effect**

    - **`ChooseStartingPokemonEffect` Effect**
        - `type`: `GamePhaseEffects.CHOOSE_STARTING_POKEMON_EFFECT`
        - **Purpose:** Manages the part of game setup where players select their Active and Benched Pokémon.
        - **Constructor Parameters:**
            - `player: Player`: The player choosing their Pokémon.
        - **Key Properties:** `player: Player`.

    - **`AfterAttackEffect` Effect**
        - `type`: `GamePhaseEffects.AFTER_ATTACK_EFFECT`
        - **Purpose:** An intermediary phase effect that occurs after an attack has completely finished but before the turn officially ends (e.g., before Pokémon Checkup for Special Conditions like Paralysis wearing off).
        - **Constructor Parameters:**

        - **Key Properties:** `player: Player`.

## III. Cross-Cutting Concerns / Patterns

- **`preventDefault` Mechanism:** This boolean flag on every `Effect` is the primary way the game allows for interaction and modification of standard actions. Abilities, Trainer cards, or Stadium effects can intercept an effect and set `preventDefault = true` to stop its default outcome or to implement a replacement behavior.
- **Layered Effects:** Many high-level actions (like an `AttackEffect`) are broken down into more granular sub-effects (like `ApplyWeaknessEffect`, `DealDamageEffect`, `AddSpecialConditionsEffect`). This allows for fine-tuned interaction points at various stages of an action's resolution.
- **Enum-Driven Dispatch:** The `type` property, populated from enums like `GameEffects`, `AttackEffects`, etc., is critical for the game engine to dispatch effects to the correct reducers or handlers that know how to process that specific type of event.
- **Contextual Data:** Each effect class carries specific data relevant to that event (e.g., `player`, `target`, `card`, `damage_amount`). This data is often mutable by handlers before the effect is fully resolved, allowing for dynamic modification of outcomes.

## IV. How to Add New Effects (General Pattern)

1.  **Identify Scope:** Determine if the new effect is a general game mechanic, part of an attack, a check, a game phase transition, or related to playing a card. This will guide which `.ts` file (and corresponding enum) it belongs to.
2.  **Define Enum Value:** Add a new string constant to the appropriate `Effects` enum (e.g., `NEW_AWESOME_EFFECT` in `GameEffects`).
3.  **Create Class:** Define a new class that implements the base `Effect` interface.
    ```typescript
    export class NewAwesomeEffect implements Effect {
      readonly type: string = GameEffects.NEW_AWESOME_EFFECT; // From the enum
      public preventDefault = false;
      // Add specific properties relevant to this new effect
      public player: Player;
      public someValue: number;

      constructor(player: Player, someValue: number) {
        this.player = player;
        this.someValue = someValue;
      }
    }
    ```
4.  **Integrate Dispatch:** In the game logic where this new event occurs, create an instance of `NewAwesomeEffect` and dispatch it through the game's effect resolution system (e.g., `store.dispatch(new NewAwesomeEffect(player, 42))`).
5.  **Implement Handlers/Reducers:** Create or update parts of the system that need to react to or modify `NewAwesomeEffect`. These handlers will check `effect.type` and then cast the effect to `NewAwesomeEffect` to access its specific properties.
```
