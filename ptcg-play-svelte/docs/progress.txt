Twinleaf Local Svelte progress

Goal
- Build a lightweight Svelte 5 local self-play frontend.
- Import two decks, run ptcg-server headless game logic in manual prompt mode, render a smooth 2D board, support core actions/prompts, and validate behavior with action-level tests.

Architecture decisions
- Keep the existing React app untouched.
- Create a separate ptcg-play-svelte app.
- Keep Node-only ptcg-server headless/card-registration code out of the browser bundle.
- Run a tiny local HTTP engine service in the Svelte package instead of auth/lobby/socket infrastructure.
- Browser UI talks to the engine through plain JSON GameView/PromptView objects.
- Svelte components should not import ptcg-server types or classes.
- Use manual prompt mode for player agency.
- Include friendly prompt UIs for common prompt types plus a JSON fallback so rare card-specific prompts do not permanently block a game.

Known gaps
- Prompt controls are intentionally generic; rare cards may still need the Advanced JSON fallback until their prompt shape is promoted into a friendly control.
- The board is a lightweight 2D implementation, not yet feature-complete polish against every React drag animation and zone affordance.
- No auth, lobby, matchmaking, deck persistence, network rooms, or React migration layer are included by design.
- Full ptcg-server test suite still has unrelated base64 expectation failures under the current Node/runtime; do not treat that as this frontend's pass/fail signal.
