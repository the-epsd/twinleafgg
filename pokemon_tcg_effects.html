Pokémon TCG Game Effects

Introduction to Special Conditions

Special Conditions are temporary states that can affect an Active Pokémon, hindering its ability to battle. They are a core mechanic in the Pokémon Trading Card Game (TCG).

  • Target: Only Active Pokémon can be affected by Special Conditions.
  • Removal: Generally, Special Conditions are removed if the Pokémon:
    • Moves to the Bench (retreats or is switched out).
    • Evolves or devolves.
    • Is affected by a specific card effect (e.g., a Trainer card like "Full Heal" or an Ability).
  • Stacking:
    • Asleep, Paralyzed, and Confused are mutually exclusive. Applying one of these will remove any existing one from this group.
    • Poisoned and Burned can coexist with each other and with Asleep, Paralyzed, or Confused.
  • Pokémon Checkup: This is a phase between turns where effects of certain Special Conditions (Poisoned, Burned, Asleep/Paralyzed recovery check) are resolved. The order is typically: Poisoned, Burned, then Asleep/Paralyzed checks.

I. Standard Special Conditions

These are the five primary Special Conditions you'll encounter most frequently.

1. Asleep

Description: An Asleep Pokémon cannot attack or retreat.

Mechanism: Indicated by turning the Pokémon card counter-clockwise. Between each player's turn (during Pokémon Checkup), the owner of the Asleep Pokémon flips a coin.

  • Heads: The Pokémon wakes up and is no longer Asleep.
  • Tails: The Pokémon remains Asleep.

Infliction: Typically caused by attack effects (e.g., an attack stating "The Defending Pokémon is now Asleep") or certain Abilities.

Removal:

  • Flipping heads during Pokémon Checkup.
  • Moving the Pokémon to the Bench.
  • Evolving or devolving the Pokémon.
  • Using specific Trainer cards or Abilities that remove Special Conditions.

Interaction: Mutually exclusive with Paralyzed and Confused. Can coexist with Burned and Poisoned.

Code Example (Conceptual):


class Pokemon:
    def __init__(self, name):
        self.name = name
        self.is_asleep = False
        # ... other conditions like is_paralyzed, is_confused

    def attempt_attack(self, target):
        if self.is_asleep:
            print(f"{self.name} is Asleep and cannot attack!")
            return False
        # ... attack logic
        print(f"{self.name} attacks {target.name}!")
        return True

    def attempt_retreat(self):
        if self.is_asleep:
            print(f"{self.name} is Asleep and cannot retreat!")
            return False
        # ... retreat logic
        self.is_asleep = False # Assuming retreat removes conditions
        print(f"{self.name} retreats.")
        return True

    def pokemon_checkup(self):
        if self.is_asleep:
            print(f"Checking if {self.name} wakes up...")
            if flip_coin() == "heads":
                print(f"{self.name} woke up!")
                self.is_asleep = False
            else:
                print(f"{self.name} is still Asleep.")

    def set_asleep(self):
        print(f"{self.name} is now Asleep!")
        self.is_asleep = True
        self.is_paralyzed = False # Mutually exclusive
        self.is_confused = False # Mutually exclusive

def flip_coin():
    import random
    return random.choice(["heads", "tails"])

2. Burned

Description: A Burned Pokémon takes damage between turns.

Mechanism: Indicated by placing a "Burn marker" on the Pokémon. During Pokémon Checkup (after each player's turn):

  1. Place 2 damage counters (20 damage) on the Burned Pokémon. (Some card effects can modify this damage).
  2. The owner of the Burned Pokémon then flips a coin.
    • Heads: The Pokémon is no longer Burned (remove the Burn marker).
    • Tails: The Pokémon remains Burned.

Historical Note: Prior to the Sun & Moon series, on a heads flip, the Pokémon would not take damage but would remain Burned. The current rule is damage first, then flip to cure.

Infliction: Caused by attacks (e.g., "The Defending Pokémon is now Burned") or Abilities.

Removal:

  • Flipping heads during Pokémon Checkup (after taking burn damage).
  • Moving the Pokémon to the Bench.
  • Evolving or devolving the Pokémon.
  • Using specific Trainer cards or Abilities.

Interaction: Can coexist with any other Special Condition.

Code Example (Conceptual):


class Pokemon:
    def __init__(self, name):
        self.name = name
        self.hp = 100
        self.damage_counters = 0
        self.is_burned = False
        self.burn_damage_per_turn = 20 # Standard burn damage

    def receive_damage(self, amount):
        self.damage_counters += amount // 10
        print(f"{self.name} took {amount} damage. Total damage: {self.damage_counters * 10}")
        if self.damage_counters * 10 >= self.hp:
            print(f"{self.name} is Knocked Out!")

    def pokemon_checkup(self):
        if self.is_burned:
            print(f"{self.name} is Burned!")
            self.receive_damage(self.burn_damage_per_turn)
            if (self.damage_counters * 10) < self.hp: # Only flip if not Knocked Out
                print(f"Checking if {self.name} recovers from Burn...")
                if flip_coin() == "heads":
                    print(f"{self.name} recovered from the Burn!")
                    self.is_burned = False
                else:
                    print(f"{self.name} is still Burned.")

    def set_burned(self, damage_amount=20):
        print(f"{self.name} is now Burned!")
        self.is_burned = True
        self.burn_damage_per_turn = damage_amount

3. Confused

Description: A Confused Pokémon may hurt itself when trying to attack.

Mechanism: Indicated by turning the Pokémon card upside down (180 degrees). When the Confused Pokémon tries to attack, its owner must flip a coin:

  • Heads: The attack works as normal.
  • Tails: The attack fails, and the Pokémon does 30 damage (3 damage counters) to itself.

Infliction: Caused by attack effects or Abilities.

Removal:

  • Moving the Pokémon to the Bench. (Unlike Asleep/Paralyzed, there's no automatic recovery attempt between turns for Confusion).
  • Evolving or devolving the Pokémon.
  • Using specific Trainer cards or Abilities.

Interaction: Mutually exclusive with Asleep and Paralyzed. Can coexist with Burned and Poisoned. Retreating while Confused is normal; no coin flip is required for the retreat itself.

Code Example (Conceptual):


class Pokemon:
    def __init__(self, name):
        self.name = name
        self.hp = 100
        self.damage_counters = 0
        self.is_confused = False

    def receive_damage(self, amount):
        self.damage_counters += amount // 10
        print(f"{self.name} took {amount} damage. Total damage: {self.damage_counters * 10}")
        if self.damage_counters * 10 >= self.hp:
            print(f"{self.name} is Knocked Out!")

    def attempt_attack(self, target):
        if self.is_confused:
            print(f"{self.name} is Confused. Flipping a coin to attack...")
            if flip_coin() == "tails":
                print(f"{self.name} hurt itself in its confusion!")
                self.receive_damage(30)
                return False # Attack fails
            else:
                print(f"{self.name} attacks successfully through confusion!")
        # ... proceed with attack logic
        print(f"{self.name} attacks {target.name}!")
        return True

    def set_confused(self):
        print(f"{self.name} is now Confused!")
        self.is_confused = True
        self.is_asleep = False # Mutually exclusive
        self.is_paralyzed = False # Mutually exclusive

4. Paralyzed

Description: A Paralyzed Pokémon cannot attack or retreat.

Mechanism: Indicated by turning the Pokémon card clockwise. Paralysis lasts for only one turn. At the end of the player's next turn (during Pokémon Checkup), the Pokémon recovers from Paralysis automatically.

Infliction: Caused by attack effects or Abilities.

Removal:

  • Automatically at the end of the affected player's next turn (during their Pokémon Checkup phase).
  • Moving the Pokémon to the Bench.
  • Evolving or devolving the Pokémon.
  • Using specific Trainer cards or Abilities.

Interaction: Mutually exclusive with Asleep and Confused. Can coexist with Burned and Poisoned.

5. Poisoned

Description: A Poisoned Pokémon takes damage between turns.

Mechanism: Indicated by placing a "Poison marker" on the Pokémon. During Pokémon Checkup (after each player's turn), place 1 damage counter (10 damage) on the Poisoned Pokémon. Some card effects can increase the amount of damage taken from Poison.

Infliction: Caused by attack effects or Abilities (e.g., "The Defending Pokémon is now Poisoned").

Removal:

  • Moving the Pokémon to the Bench.
  • Evolving or devolving the Pokémon.
  • Using specific Trainer cards or Abilities.
  • (Unlike Burned, there is no coin flip to recover from Poison; it persists until actively removed).

Interaction: Can coexist with any other Special Condition.

Code Example (Conceptual):


class Pokemon:
    def __init__(self, name):
        self.name = name
        self.hp = 100
        self.damage_counters = 0
        self.is_poisoned = False
        self.poison_damage_counters = 1 # Standard is 1 damage counter (10 damage)

    def receive_damage_counters(self, num_counters):
        self.damage_counters += num_counters
        print(f"{self.name} took {num_counters * 10} damage. Total damage: {self.damage_counters * 10}")
        if self.damage_counters * 10 >= self.hp:
            print(f"{self.name} is Knocked Out!")

    def pokemon_checkup(self):
        if self.is_poisoned:
            print(f"{self.name} is Poisoned! It takes {self.poison_damage_counters * 10} damage.")
            self.receive_damage_counters(self.poison_damage_counters)

    def set_poisoned(self, num_counters=1):
        print(f"{self.name} is now Poisoned!")
        self.is_poisoned = True
        self.poison_damage_counters = num_counters

II. Other Conditions

These conditions are less common and have specific mechanics, often tied to particular Pokémon cards.

1. Imprisoned

Description: An Imprisoned Pokémon cannot use its Poké-Powers or Poké-Bodies (older mechanics similar to Abilities).

Mechanism: Indicated by an "Imprisoned marker." This condition is very rare.

Infliction: Caused by the effect of a specific Pokémon's attack or Ability.

Removal:

  • Evolving the Imprisoned Pokémon.
  • Using a card effect that removes any condition.
  • The Pokémon leaving play.
  • Notably, retreating or switching out does NOT remove the Imprisoned condition.

Interaction: Traditionally affected older power types (Poké-Powers/Bodies).

Code Example (Conceptual):


class Pokemon:
    def __init__(self, name, has_poke_power=False, has_poke_body=False):
        self.name = name
        self.is_imprisoned = False
        self.has_poke_power = has_poke_power
        self.has_poke_body = has_poke_body

    def use_poke_power(self):
        if self.is_imprisoned:
            print(f"{self.name} is Imprisoned and cannot use its Poké-Power!")
            return False
        if not self.has_poke_power:
            print(f"{self.name} does not have a Poké-Power.")
            return False
        print(f"{self.name} uses its Poké-Power!")
        return True

    def check_poke_body_active(self):
        if self.is_imprisoned and self.has_poke_body:
            print(f"{self.name}'s Poké-Body is negated by Imprisonment!")
            return False
        if self.has_poke_body:
            print(f"{self.name}'s Poké-Body is active.")
            return True
        return False

    def set_imprisoned(self):
        print(f"{self.name} is now Imprisoned!")
        self.is_imprisoned = True

    def evolve(self):
        # ... evolution logic
        self.is_imprisoned = False # Imprisoned removed on evolution
        print(f"{self.name} evolved and is no longer Imprisoned.")

2. Shock-wave

Description: By itself, the Shock-wave condition does nothing. However, specific attacks may have powerful effects against Pokémon with this condition.

Mechanism: Indicated by a "Shock-wave marker." This is another very rare condition.

Infliction: Caused by the effect of a specific Pokémon's attack.

Removal:

  • Evolving the affected Pokémon.
  • The Pokémon leaving play.
  • Using a card effect that removes any condition.
  • Notably, retreating or switching out does NOT remove the Shock-wave condition.

Interaction: This condition is a passive marker whose sole purpose is to be recognized by other card effects.

Code Example (Conceptual):


class Pokemon:
    def __init__(self, name):
        self.name = name
        self.has_shock_wave_marker = False
        self.hp = 100
        self.damage_counters = 0


    def set_shock_wave_marker(self):
        print(f"{self.name} now has a Shock-wave marker.")
        self.has_shock_wave_marker = True

    def remove_shock_wave_marker(self):
        self.has_shock_wave_marker = False
        print(f"{self.name} no longer has a Shock-wave marker.")

    def evolve(self):
        # ... evolution logic
        self.remove_shock_wave_marker() # Shock-wave removed on evolution
        print(f"{self.name} evolved and lost its Shock-wave marker.")

class AttackerPokemon(Pokemon):
    def special_attack(self, target_pokemon):
        if target_pokemon.has_shock_wave_marker:
            print(f"{self.name}'s special attack instantly Knocks Out {target_pokemon.name} due to Shock-wave marker!")
            # Simulate Knock Out
            target_pokemon.damage_counters = target_pokemon.hp // 10
            print(f"{target_pokemon.name} is Knocked Out!")
        else:
            print(f"{self.name}'s special attack has no extra effect on {target_pokemon.name}.")
